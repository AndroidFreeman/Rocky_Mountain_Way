# xue：极坐标螺旋轨迹生成（`rocky_mountain_way.thr`）思路说明

## 目标

- 输入：一张灰度/彩色图片（脚本内部按灰度处理）
- 输出：一条从中心向外扩展的螺旋轨迹，保存为 `rocky_mountain_way.thr`（每行 `theta rho`）
- 预览：将 `(theta, rho)` 轨迹绘制成极坐标图，保存为 `rocky_mountain_way_preview.png`

核心想法：用“等间距的阿基米德螺旋”扫描整张图，在扫描的过程中根据像素的“黑度”对半径做周期性扰动（wobble），让轨迹在黑的区域更“摆动”，从而形成可视的明暗纹理。

如果用一句话概括：**把“图像灰度”映射成“螺旋半径的微小抖动幅度”，用单条连续轨迹表达纹理。**

## 你答辩怎么讲（2 分钟版本）

1. 我先把输入图转成稳定的灰度场（裁成正方形 + 归一化 + 去噪）。
2. 我生成一条从中心向外的阿基米德螺旋，用极坐标参数化轨迹点。
3. 每个轨迹点映射到图像坐标采样灰度，得到黑度 `darkness`。
4. 黑度越大，我给半径加一个周期扰动 `wobble`（正弦项），相当于在黑色区域“加密线条”。
5. 把每个点写成 `.thr`（`theta rho`），并画一张极坐标预览图，便于展示效果。

## 数学/公式层面的表述（老师爱听）

设轨迹参数为离散步 `i=0..N-1`。

- 极角：`theta_i = (i/N) * (2π * num_spirals)`
- 基础半径（归一化）：`rho_i = i/N`（中心→外圈线性增长，阿基米德螺旋）

将 `(theta_i, rho_i)` 映射到像素：

- 像素半径：`r_pixel = rho_i * (size/2 - 1)`
- `x = cx + r_pixel * cos(theta_i)`
- `y = cy - r_pixel * sin(theta_i)`

这里 **y 用减号** 是因为 OpenCV 图像坐标 y 轴向下为正。

像素黑度（0~1）：

- `darkness = 1 - I(y,x)/255`

半径扰动：

- `wobble = A * darkness * sin(freq * theta)`
- `A = 0.8 * (1/num_spirals)`（用螺旋间距约束幅度，避免跨圈过多）

最终半径：

- `rho'_i = clip(rho_i + wobble, 0, 1)`

## 输入预处理（让图像更稳定）

对应代码在 [xue.py](file:///d:/Tree_Code/Rocky_Mountain_Way/xue.py)。

- **灰度读取**：`cv2.imread(..., IMREAD_GRAYSCALE)`
- **裁剪成正方形**：取短边 `size=min(h,w)`，从中心裁切，保证极坐标映射对称。
- **归一化**：`cv2.normalize` 把像素拉到 0~255，减少曝光差异带来的影响。
- **阈值化 + 模糊**：先二值化再高斯模糊，用于抑制噪声，让“黑度”更平滑（轨迹不会抖得太碎）。

答辩用解释：

- 预处理的目标不是“识别物体”，而是获得一个**稳定的强弱纹理场**，保证轨迹的扰动不会被噪声主导。

## 螺旋参数（轨迹密度怎么来的）

- `num_spirals`：螺旋圈数（越大轨迹越密，细节越多）
- `steps_per_spiral`：每圈采样点数（越大轨迹越平滑、文件越大、计算越慢）
- `total_steps = num_spirals * steps_per_spiral`
- `max_theta = num_spirals * 2π`：角度从 0 扫到 `max_theta`

每个采样点 `i`：

- `theta = (i / total_steps) * max_theta`
- `base_rho = i / total_steps`（0→1，代表从中心向外的归一化半径）

这里用 `base_rho` 让轨迹半径随 `theta` 线性增长，相当于阿基米德螺旋。

调参建议（你可以按这个顺序现场讲）：

- 想更细：先增大 `num_spirals`（轨迹更密）
- 想更平滑：再增大 `steps_per_spiral`（采样更密）
- 想纹理更明显：调大 `fixed_freq` 或调大扰动上限（但要注意跨圈）

## 从极坐标映射到像素坐标（以及“坐标系反转”修复）

为了从图片上读取该点的灰度，需要把 `(theta, base_rho)` 转成像素坐标 `(x, y)`。

- 把归一化半径换成像素半径：
  - `r_pixel = base_rho * (size/2 - 1)`
- `x = center_x + r_pixel * cos(theta)`
- `y = center_y - r_pixel * sin(theta)`

重点：**y 这里是减号**。

- 数学坐标系：y 轴向上为正
- 图像坐标系（OpenCV）：y 轴向下为正

所以必须用 `- sin(theta)` 来翻转，否则预览会在上下方向镜像。

答辩可补一句：这个问题本质是“图像坐标系”和“数学坐标系”的约定不同，属于常见工程坑。

## 黑度→半径扰动（让轨迹“画出”明暗）

定义黑度：

- 像素越黑，`img[y,x]` 越小
- `darkness = 1 - img[y,x]/255`，范围 0~1（越黑越接近 1）

然后用一个固定频率的正弦扰动：

- `wobble = max_amplitude * darkness * sin(fixed_freq * theta)`（只在 `darkness>0.1` 时启用）

其中：

- `fixed_freq`：扰动频率（越大纹理越“密/细”）
- `track_distance = 1/num_spirals`：相邻螺旋轨道的理论间距
- `max_amplitude = track_distance * 0.8`：扰动最大幅度，控制“不跨越相邻圈”的安全范围

最终：

- `final_rho = clip(base_rho + wobble, 0, 1)`

直觉解释：

- 白的地方（darkness≈0）：几乎没有摆动，轨迹趋近标准螺旋
- 黑的地方（darkness→1）：摆动幅度变大，轨迹局部“抖动”，形成更密的线条/纹理，从视觉上更“黑”

为什么用“正弦扰动”？

- 正弦是连续且可控的周期函数，输出轨迹平滑、便于设备执行。
- 频率控制纹理粗细，幅度控制明暗强弱，讲起来也清楚。

## 输出格式（`.thr`）

脚本把每个点写成一行：

- `theta rho`
- 均保留 4 位小数

文件：`rocky_mountain_way.thr`

如果后续有机械臂/绘图设备，一般只要按顺序读取这些点，把 `(theta,rho)` 再映射回设备坐标即可。

答辩补充：使用极坐标保存的好处是“设备坐标系”可灵活替换，`rho` 可以再映射到不同尺度。

## 复杂度与展示要点（答辩可用）

- 复杂度：`O(total_steps)`，主要成本在循环采样与写文件。
- 鲁棒性：中心裁剪 + 归一化 + 模糊，让不同光照/尺寸的输入更稳定。
- 关键创新点：把图像明暗映射为螺旋轨迹的“径向扰动”，用单条连续轨迹表达灰度纹理。
- 关键 bugfix 点：图像坐标系 y 轴方向与数学坐标系相反，必须用 `-sin(theta)`。

常见失败场景（提前准备一句话）：

- 输入图太黑/太白：`darkness` 分布极端，纹理会不明显；可调阈值/对比度或换输入。
- `num_spirals` 太小：轨迹稀疏，看不出细节；太大：计算和文件变大。
- 幅度太大：跨圈导致线条混乱；所以用 `1/num_spirals` 来约束最大幅度。

## 面试式 Q&A（老师追问可用）

**Q1：为什么不是用 Canny / 边缘提取？**

- 这里目标不是提轮廓，而是用“连续轨迹”表达明暗纹理；螺旋扫描更适合输出单笔连续路径。

**Q2：为什么要裁正方形？**

- 极坐标天然以圆为对称，如果长宽不同，外圈会不均匀变形，影响观感和解释一致性。

**Q3：为什么 `rho` 归一化到 0~1？**

- 归一化后轨迹与分辨率解耦，后续映射到任意画布/设备尺寸更方便。
